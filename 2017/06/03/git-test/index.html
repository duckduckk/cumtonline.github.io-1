git测试

6月 03 2017

大约是5月中旬对后端、移动组(除了请假的同学)进行了git测试。

前端组尚未测。本学期结束前应该会补上。

题目整理如下

1.如何(什么没命令)把一个目录(文件夹)变成git可以管理的仓【初始化仓库】

git init

2.把文件添加到仓库的命令

git add

3.把文件提交到仓库的命令

git commit

4.查看当前仓库状态的命令(查看文件是否被修改)

git status

5.查看修改内容的命令

git diff

6.git中HEAD指针指向的是什么

HEAD(大写)是”current branch”(当下的分支)。更多请戳这儿

7.查看提交历史的命令

git log

8.查看命令历史的命令

git reflog

9.git push origin master的作用是？此处origin是指什么？这个origin是可以修改的吗？

本地当前分支的内容推送到远程仓库;远程仓库;可以

10.git的工作区和暂存区分别是指什么？和git add以及git commit 的关系是？

戳这儿

11.git fetch 和 git pull的区别,谁的安全性更高

git fetch不会自动merge。git fetch安全性更高。

12.git branch 和 git branch < name > 的区别

git branch查看分支

git branch < name > 创建分支

13.如何切换分支,如何合并分支

git checkout
git merge
